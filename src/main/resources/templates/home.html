Endpoints REST da Aplicação

1. PessoaController (`/api/pessoas`)

POST
/cadastro

Descrição: Endpoint REST para cadastrar um novo usuário (Completo).

URL Completa: http://localhost:8080/api/pessoas/cadastro

Body Requerido: PessoaRequest (com todos os campos).

Resposta de Sucesso: Status 201 (Created) e o PessoaResponse.

POST
/cadastro/simplificado

Descrição: Endpoint REST para cadastrar um novo usuário com apenas e-mail e senha. Utiliza placeholders e inicia o processo de ativação.

URL Completa: http://localhost:8080/api/pessoas/cadastro/simplificado

Body Requerido: CadastroSimplificadoRequest (email e senha).

Resposta de Sucesso: Status 201 (Created) e o PessoaResponse.

2. AtivacaoController (`/api/v1/ativacao`)

GET
/

Descrição: Lógica para validar o token de ativação (Redis) e ativar a conta (PostgreSQL). Este é o link clicado pelo utilizador no e-mail.

Exemplo de URL: http://localhost:8080/api/v1/ativacao?token=a1b2c3d4e5f6...

Parâmetro: token (Query Parameter, obrigatório).

Resposta: Status 200 (OK) em caso de sucesso ou Status 400 (Bad Request) se o token for inválido/expirado.
